# Логирование в проекте carl-superchat

В данном проекте реализована централизованная система логирования, которая объединяет вывод стандартной консоли и кастомных логгеров в текстовые файлы с разделением по датам.

## Основные компоненты

### 1. Модуль `lib/logger.js`
Ядро системы логирования. Расположение: `server/lib/logger.js`

Основные функции:
- **`createLogger(context)`**: Создает объект логгера с методами `info`, `warn`, `error`, привязанными к определенному контексту (например, `DB`, `SERVER`).
- **`writeToFile(level, context, message)`**: Низкоуровневая функция записи в файл.
- **Ротация по датам**: Логи записываются в файлы формата `combined-YYYY-MM-DD.log`. Ошибки дублируются в `error-YYYY-MM-DD.log`.

### 2. Перехват `console` в `server.js`
При запуске сервера стандартные методы `console.log`, `console.warn` и `console.error` подменяются. Это гарантирует, что любое использование `console` (даже в сторонних библиотеках или подпакетах) попадет в файлы логов.
- `console.log` -> Записывается как `[INFO] [SERVER]`
- `console.warn` -> Записывается как `[WARN] [SERVER]`
- `console.error` -> Записывается как `[ERROR] [SERVER]`

## Расположение логов
Все логи сохраняются в директорию `logs/` в корне проекта (директория добавлена в `.gitignore`).

- **`logs/combined-YYYY-MM-DD.log`**: Содержит все записи (INFO, WARN, ERROR).
- **`logs/error-YYYY-MM-DD.log`**: Содержит только записи уровня ERROR.

## Формат записи
Запись в файле имеет следующий вид:
`ISO_TIMESTAMP [LEVEL] [CONTEXT] Message {JSON_OR_STACK_TRACE}`

Пример:
`2026-03-26T12:00:00.000Z [INFO] [SERVER] UI: http://localhost:3301`

## Использование

### Стандартный способ (рекомендуется)
Просто используйте `console`, как обычно. Все сообщения будут автоматически сохранены в логи.
```javascript
console.log('Информационное сообщение');
console.error(new Error('Критическая ошибка'));
```

### Кастомный контекст
Если нужно выделить логи конкретного модуля:
```javascript
const { createLogger } = require('./lib/logger');
const logger = createLogger('MY-MODULE');

logger.info('Сообщение в контексте модуля');
// В логе будет: [INFO] [MY-MODULE] Сообщение в контексте модуля
```

### Логирование с клиента (Test UI)

Логи из браузерного интерфейса (`server/public/index.html`) автоматически отправляются на сервер через API endpoint `POST /api/logs`:

```javascript
// В браузере - console.log автоматически отправляется на сервер
console.log('Сообщение из UI');
console.warn('Предупреждение в UI');
console.error('Ошибка в UI', error);
```

Клиентский перехватчик:
- Перехватывает `console.log`, `console.warn`, `console.error`
- Отправляет логи на сервер с контекстом `TEST-UI`
- Использует batching (накопление и отправка пачками с интервалом 500мс)
- Ошибки отправляются немедленно
- Перехватывает необработанные ошибки (`window.onerror`, `unhandledrejection`)

## Технические детали
- **Объекты**: Все объекты, переданные в логгер, автоматически сериализуются в JSON.
- **Ошибки**: При передаче объекта `Error` сохраняется стек вызова (stack trace).
- **Вывод**: Логи пишутся в файлы синхронно (`appendFileSync`), что гарантирует запись даже при падении процесса, но может иметь небольшое влияние на производительность при экстремально больших объемах логов.
- **Консоль**: Параллельно с записью в файл, логи выводятся в `stdout` (INFO, WARN) и `stderr` (ERROR).
- **Клиентские логи**: Логи из браузера отправляются через `fetch('/api/logs')` с batching (накопление и отправка пачками с интервалом 500мс).
